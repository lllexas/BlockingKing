# Gameplay 设计原理 - 技术策划 Demo

## 一句话定位

BlockingKing 是一个把推箱子的空间思考，放进卡牌战棋肉鸽流程里的原型。

它想解决的问题不是“怎么把推箱子做得更难”，而是：

```text
如何做一款不让当代玩家畏难的推箱子游戏？
```

传统推箱子的压力很明确：

```text
一步错，可能全盘错
玩家不知道自己是不是已经死局
重开会让前面的思考显得白费
高难关卡容易让玩家直接退出
```

所以本项目的核心不是削弱推箱子，而是给玩家更多承接压力的方式：

```text
推得好，有更高收益
推不动，可以进入战斗阶段
走错了，可以悔棋，但要付资源代价
局面太复杂，可以用卡牌修正
不想打这一关，可以 Skip 拿低保
战后可以进 Shop / Event 调整构筑
动作和音乐保持律动，让推箱子的思考不那么干硬
```

## 玩家体验总览

玩家进入的不是单个孤立关卡，而是一局连续推进的 Run。

每一轮，玩家先做主战斗选择：

```text
Classic  经典推箱子关
Escort   护送关
Skip     跳过本轮，拿较低补偿
```

完成主战斗后，再进入战后选择：

```text
Shop     商店
Event    随机事件
```

局内资源包括：

```text
HP
金币
卡组
手牌
悔棋次数和代价
```

这让玩家不是“一关卡死就结束”，而是在一局 Run 中不断做选择：

```text
这一关要不要继续动脑
什么时候结算当前成果
要不要花卡牌救局
要不要用金币或血量悔棋
战后是买卡、拿事件，还是保存资源
```

推箱子仍然是核心，但失败压力被拆进了资源、战斗、构筑和选择里。

## 基础推箱子规则

基础规则仍然是推箱子：

```text
玩家在格子地图中移动
箱子可以被推动
墙和障碍阻挡移动
目标点用于判定箱子归位
玩家需要规划站位、推动方向和通路
```

在这个基础上，项目加入了两类东西。

压力来源：

```text
敌人
HP
生存失败条件
Classic 阶段二刷怪压力
Escort 核心箱被摧毁压力
```

补救和成长手段：

```text
卡牌
金币奖励
悔棋
Shop / Event
Skip
肉鸽选择
```

它们不是替代推箱子，而是让玩家在推箱子卡住时，不只剩下重开。

## Classic：先纯推箱，后卡牌战斗兜底

Classic 是经典推箱子关。

它分成两个阶段。

### 阶段一：纯推箱探索 / 预结算

进入 Classic 后：

```text
卡牌不能用
不会立刻刷怪
玩家只能移动和推箱子
行动不限步数
UI 提供一个【结算】按钮
玩家随时可以点【结算】
```

点【结算】时，系统按照“当前已经推到目标点上的箱子数”发金币。

阶段一金币规则：

```text
第 1 个箱子：3 金
第 2 个箱子：6 金
第 3 个箱子：9 金
第 n 个箱子：3 * n 金
```

这一阶段鼓励玩家尽量先靠纯操作把箱子推进目标点。

玩家推得越多，单个箱子的收益越高。

这不是强迫玩家完美通关，而是告诉玩家：

```text
你愿意多动脑，就能拿更高收益
你不想继续烧脑，也可以先结算
```

### 阶段切换

玩家点击【结算】后：

```text
播放简单转场
记录阶段一已经结算过的箱子
发放阶段一金币
卡牌开始可用
```

同时，当前还没有被箱子占据的普通目标点，会变成敌人刷新目标点。

也就是说：

```text
阶段一已经被箱子占住的目标点 = 安全
阶段一还空着的目标点 = 阶段二目标点兼刷怪点
```

玩家越早结算，阶段二留下的刷怪点越多。

玩家越努力把箱子先推上目标点，阶段二压力越小。

这里形成一个很清楚的选择：

```text
继续纯推箱 = 高收益，少刷怪点
提前结算 = 低收益，多刷怪点，但可以用卡牌和战斗兜底
```

### 阶段二：卡牌 + 战斗完成阶段

阶段二开始后：

```text
玩家可以使用卡牌
尚未被箱子压住的目标点会持续刷新敌人
玩家需要继续推动剩余箱子到目标点
玩家需要消灭所有敌人
```

阶段二成功条件：

```text
所有箱子都在目标点上
并且场上没有存活敌人
```

阶段二失败条件：

```text
玩家死亡
```

阶段二完成后，再按照“第二阶段新推进目标点的箱子数”发金币。

注意：阶段一已经结算过的箱子不重复计算。

阶段二金币规则：

```text
第 1 个箱子：3 金
第 2 个箱子：4 金
第 3 个箱子：5 金
第 n 个箱子：n + 2 金
```

因此 Classic 的完整体验是：

```text
先给玩家一段低压纯推箱时间
愿意动脑，就在阶段一拿高收益
不想继续精算，就结算进入阶段二
阶段二用卡牌、战斗和操作补完剩余局面
但阶段二收益更低，敌人压力更高
```

Classic 的价值，是把“必须一次性完美解完”改成“推多少、结算多少、再决定要不要进入兜底阶段”。

## Escort：在敌人压力下护送核心箱

Escort 是护送关。

它不要求玩家解完整张静态谜题，而是要求玩家把关键箱子护送到终点。

进入 Escort 后：

```text
手牌可用
存在次要奖励箱和次要目标点
敌人会在部分次要目标点不断刷新
地图中存在核心箱，也就是必须护送的箱子
地图中存在核心目标点，也就是核心箱的终点
核心箱与玩家共享血量，且仇恨高于玩家
```

Escort 成功条件：

```text
跨越整张地图把核心箱推到核心目标点
```

Escort 失败条件：

```text
玩家死亡
核心箱被摧毁
```

Escort 奖励规则：

```text
核心箱到达核心目标点后，发放护送完成金币
次要奖励箱如果被推到次要目标点上，会作为奖励箱计入金币
完成后还可以从 Escort 完成奖励池中抽取额外奖励
```

Escort 的压力来自敌人持续干扰。

玩家不能只盯着箱子，也不能只打怪。

玩家每回合都要判断：

```text
这一回合先推核心箱，还是先清持续刷新的敌人
核心箱会不会被打爆
玩家自己会不会被围住
有没有机会顺手拿普通奖励箱
要不要用卡牌把核心箱一次推远
```

所以 Escort 的体验不是“把所有箱子慢慢推到位”。

它更像一个带推箱目标的护送战斗：

```text
核心箱是任务目标
敌人是时间压力
卡牌是推进和解围工具
普通奖励箱是额外收益
```

在 Escort 中，长距离推箱牌非常关键。

如果玩家只能一格一格推核心箱，就容易被敌人围住。

战车、主教、皇后这类牌可以在单个回合内把核心箱推进一大段距离。

它们的作用不是单纯打伤害，而是给护送过程创造突破窗口。

核心箱还有一个重要机制：位移护盾。

核心箱被推动后，会根据移动距离获得护盾。

简单理解就是：

```text
推得越远，箱子越能抗
停得越久，护盾越会衰减
```

所以 Escort 不是“把箱子放在原地等安全”。

玩家需要不断制造推进：

```text
推核心箱前进
核心箱因为位移获得护盾
护盾帮核心箱扛住敌人压力
如果太久不推，护盾开始衰减
敌人压力重新变危险
```

这个机制让“推箱子”本身变成防御手段。

玩家不是先打完所有敌人再慢慢推箱。

更好的节奏是：

```text
推进核心箱
获得护盾
利用护盾窗口清敌或调整站位
再继续推进
```

这也进一步提高了长距离推箱牌的价值。

一次把核心箱推得更远，不只是节省回合，也是在给核心箱叠更厚的生存预算。

## 悔棋：允许后悔，但要付资源代价

项目加入了关卡内悔棋机制。

玩家每走错一步，都不是立刻完蛋。

只要还有历史记录，就可以选择悔棋。

悔棋会把场面退回上一次行动前，包括：

```text
角色、箱子、敌人的位置和状态
手牌、牌库和弃牌堆
关卡内已经发生的阶段状态
结算相关记录
```

玩家可以通过 UI 悔棋。

当前有两种支付方式：

```text
金币悔棋
血量悔棋
```

代价随本关已付费悔棋次数增长：

```text
金币代价 = 第 N 次付费悔棋的 N²
血量代价 = 已付费悔棋次数 × 2 + 1
```

也就是说：

```text
第一次金币悔棋 1 金币
第二次金币悔棋 4 金币
第三次金币悔棋 9 金币

第一次血量悔棋 1 HP
第二次血量悔棋 3 HP
第三次血量悔棋 5 HP
```

悔棋限制：

```text
没有历史记录时不能悔棋
当前动作或演出还没播完时不能悔棋
玩家血量不足以支付时不能用血量悔棋
金币不足时不能用金币悔棋
Classic 阶段一结算后会清空阶段边界前的悔棋历史
```

悔棋的设计含义很直接：

```text
允许玩家补救一步失误
但补救不是免费的
错误会转化为金币或血量压力
```

这比直接重开更适合一局 Run 的节奏。

## 卡牌：把唯一解变成可修正局面

卡牌不是独立于推箱子的战斗技能，而是“推箱子局面的修正工具”。

基础移动只解决相邻一格的问题：

```text
空地：走过去
敌人：近身攻击
箱子：推一格
墙体：阻挡
```

卡牌把这个基础动作扩展成不同棋子的走法，让玩家在卡住时有新的处理方式：

```text
直线冲撞
斜线冲撞
短距离斜向调整
跳跃落点
翻越前排
近身范围解场
八方向万能瞄线
```

卡牌解决的不是“让玩家无视推箱子”，而是解决推箱子里最容易让玩家畏难的几类问题：

```text
站位差一格，走不到正确推箱角度
箱子被推到尴尬位置，需要额外方向修正
敌人站在推箱路径上，打断解谜思路
墙体或箱子挡住敌人，普通移动处理不到
玩家被围住，继续推箱前必须先清场
当前关卡有解，但玩家不想继续精算唯一解
```

当前已经配置并实现的卡牌如下。

| 卡牌 | 释放规则 | 当前效果 | 解决的推箱子问题 | 解决的战斗问题 |
|---|---|---|---|---|
| 战车冲撞 | 上下左右直线 | 沿直线前进；穿过空地；攻击敌人；推动箱子；攻击墙体后停止 | 处理横竖长线推箱，把箱子沿走廊快速推进，打开被墙或箱子堵住的路线 | 清理直线敌人，打到普通移动够不到的远处目标 |
| 主教冲撞 | 四个斜向直线 | 规则同战车冲撞，但方向改为斜线 | 补足横竖移动的死角，让玩家用斜线调整站位或处理斜向箱子 | 清理斜线敌人，避免所有战斗都变成横竖对齐 |
| 士兵冲撞 | 斜向相邻一格 | 向一个斜角移动一格；落点是敌人则攻击；落点是箱子则斜向推一格；落点是墙则攻击墙体 | 低成本处理“只差一个斜角”的微调局面，减少为了换站位绕大圈 | 处理贴身斜角敌人，提供便宜的小范围补刀 |
| 骑士践踏 | 日字格 | 无视路径跳到日字格；空地可落脚；敌人可攻击；墙体可攻击；箱子落点不可执行 | 越过墙、箱子和敌人形成的路径阻挡，直接跳到新的推箱角度 | 打到墙后或箱后目标，也能从包围中跳出 |
| 火炮冲撞 | 上下左右直线 | 沿直线前进；遇到第一个敌人或箱子时翻越，不伤害它；之后继续处理后排目标 | 当箱子挡在路线上时，不强行推动箱子，避免破坏当前箱子局面 | 专门处理“敌人在箱子后面”或“前排后面有高优先级目标”的局面 |
| 主帅践踏 | 自身周围 3x3 | 不位移，对周围 8 格敌人和墙体造成伤害 | 本身不直接推箱，作用是清掉贴身干扰，让玩家重新回到推箱节奏 | 被围住时的解场牌，解决近身敌人过多导致无法继续操作的问题 |
| 皇后冲撞 | 八方向直线 | 战车和主教的合并版，横竖斜八方向都能冲撞 | 万能瞄线牌，降低“我手里有牌但方向不对”的挫败 | 同时覆盖横竖和斜线清敌，是高灵活度救局牌 |

在 Classic 阶段一，手牌被锁定。

玩家先面对一段纯推箱子判断：

```text
能靠自己推多少，就拿多少高额阶段一金币
```

在 Classic 阶段二和 Escort 中，手牌开放。

这时卡牌的作用变成：

```text
把没有完成的推箱子局面继续往前推进
处理阶段二刷出的敌人
保护玩家和核心箱
把“我卡住了”转化为“我要不要花一张牌解决”
```

因此卡牌系统的重点不是技能爽感，而是降低推箱子失败感。

## 节拍与音乐：把回合动作和音乐律动对齐

玩家在关卡里能直接看到和听到的是：

```text
右上角有 BGM 面板
玩家可以点击切换曲目
每首曲子有自己的速度和拍型
玩家移动、推箱、出牌、敌人行动都会按当前音乐的速度播放
结算 UI、卡牌动画和镜头切入也会跟随同一套速度
```

这不是为了把游戏做成强制音游。

BlockingKing 不要求玩家必须在某个节拍点输入，也不会因为玩家错拍而惩罚玩家。

这套设计的出发点是：当前玩法本身已经具有明显的回合节拍性质。

每一步操作，原本就是一个回合单位：

```text
玩家观察局面
玩家提交一次移动、推箱、攻击或出牌
系统结算玩家行为
敌人和刷怪系统给出回应
场面进入下一次可判断状态
```

玩家行动和敌人行动天然存在一种“你一下，我一下”的节拍结构。

这里的设计判断是：

```text
既然关卡回合本身已经有节拍性质
而音乐也以节拍组织时间
那就可以把游戏演出和 BGM 的共同点对齐起来
```

更重要的是，人在听音乐时会本能地感知律动。

玩家不需要被告知“现在是第几拍”，也会自然感觉到音乐速度。

如果玩家动作、敌人动作、推箱动画的节奏和 BGM 岔开，就会产生别扭感：

```text
音乐在往前走
角色动作却慢半拍或快半拍
敌人回应落在不自然的位置
玩家会感觉操作反馈和听觉氛围不在一个世界里
```

这会削弱沉浸感。

所以节拍同步不是炫技功能，而是顺应玩家对音乐律动的本能。

律动在这里是一个沉浸工具：

```text
动作不和音乐打架
回合反馈更顺
玩家更容易进入当前局面
推箱子的压力被包进稳定的节奏里
```

BGM 同步系统做的不是凭空创造回合律动，而是把已经存在的回合节拍显性化、顺滑化。

玩家不是在无节奏地挪格子，而是在一拍一拍地推进局面。

## Skip / Shop / Event：给玩家离开死胡同的出口

Skip 是一个低收益保底。

它允许玩家承认：

```text
我现在不想解这一关
这个局面对当前构筑不划算
我想保存资源进入下一轮
```

Skip 不是高收益选项，但它让玩家不至于被某一关卡住。

战后 Shop / Event 提供恢复节奏。

商店让玩家把金币转化为能力：

```text
买卡
买道具
补强构筑
```

事件提供不确定收益：

```text
奖励
风险交换
资源补充
节奏变化
```

这让推箱子失败或高压之后，玩家还有喘息和选择。

## UI 与反馈：让玩家知道自己在做什么

当前 UI 服务于“让玩家知道自己在哪里”。

例如：

```text
Classic 阶段显示箱子数量和预期金币
HUD 显示 Round、HP、金币
选择面板直接展示 Classic / Escort / Skip
结算面板逐段展示奖励来源
悔棋面板显示金币和血量代价
商店显示当前金币和商品价格
BGM 面板显示当前曲目并允许切歌
```

这些反馈减少信息黑箱，让玩家知道：

```text
我正在做什么
我能得到什么
我还剩什么资源
我为什么失败或成功
```

## 系统如何共同降低畏难感

| 系统 | 玩家看到的体验 | 降低畏难的方式 |
|---|---|---|
| Classic 阶段一 | 安静推箱，随时结算 | 先给低压空间，不急着战斗 |
| Classic 阶段二 | 解锁卡牌，目标点变刷怪压力 | 允许玩家用战斗和卡牌补完剩余局面 |
| Escort | 护送核心箱到终点 | 把静态谜题改成动态局面管理 |
| 位移护盾 | 核心箱推得越远越能抗，停久会衰减 | 让推进本身成为防御，鼓励玩家主动向前 |
| 悔棋 | 可以退一步，但要付金币或血量 | 让错误可补救，但保留代价 |
| 卡牌 | 用棋子式能力改变局面 | 把唯一解变成可修正局面 |
| 节拍与音乐 | 行动反馈和 BGM 同速 | 用律动降低连续解谜的干硬感 |
| Skip | 不想打就跳过，拿低保 | 避免玩家被单关卡死 |
| Shop / Event | 战后补强或交换资源 | 给高压后一个恢复和成长窗口 |
| UI 反馈 | 奖励、代价、状态可见 | 减少信息黑箱 |

最终目标不是让推箱子失去难度，而是让玩家敢尝试、能补救、愿意继续玩。

## 技术与配置支撑

这一节给技术策划和工程评审看。

前面的玩法体验由以下系统支撑。

### Run 主循环

当前主流程是 `RoundFlow`。

它负责一局 Run 的循环：

```text
主战斗选择
Classic / Escort / Skip
战斗或跳过结算
Shop / Event
进入下一轮
```

`RunRoundConfigSO` 控制：

```text
总轮数
每轮有多少个 encounter cycle
Classic 关卡来源
Escort 生成参数
Skip 奖励池
商店商品池
事件池
Classic / Escort 交替奖励
```

这让单个推箱子关卡被放进一局 Run 的资源循环里。

### Classic 阶段切换

阶段一结算后，未被箱子占据的普通目标点会转为 `Target.Enemy`。

`Target.Enemy` 是敌人刷新目标点，会成为刷怪系统的出生点 / 刷新锚点。

这个实现把“一阶段没完成的目标”直接转化为“二阶段的刷怪压力”。

### 核心箱位移护盾

核心箱位移护盾由 `BoxBlockSystem` 处理。

当前规则是：

```text
箱子每被推动 1 格，获得 1 点 Block
默认只作用于核心箱
奖励箱是否适用由 applyToRewardBoxes 开关控制
```

护盾衰减规则：

```text
箱子被推动的当回合，BlockIdleTurns 重置为 0
箱子停 1 个完整回合，护盾不衰减
继续不动，下一次结算时护盾减半
再继续不动，护盾清零
```

敌人攻击时，`AttackSystem` 会先消耗 Block，再结算生命伤害。

这个系统把 Escort 的核心行为绑定在一起：

```text
推箱 = 前进
推箱 = 叠护盾
停太久 = 护盾衰减
敌人压力 = 逼玩家继续推进
```

### BGM 节拍同步

节拍与音乐由以下系统协作：

```text
BgmPromptSO：记录每首 BGM 的 BPM、拍型、roundTripBeat 和音量修正
BgmPlaylistSO：把多首 BGM 组织成 Run 中可切换的播放列表
BgmRecordPlayer：负责播放 BGM，并把当前曲目的节拍参数同步给运行系统
DrawSystem：根据同步来的 BPM 计算 beatDuration，控制移动、攻击、推箱等演出时长
IntentSystem：把玩家、敌人、刷怪等行为整理成按拍播放的演出步骤
BeatTiming：给卡牌动画、结算 UI、镜头 intro 等系统提供统一的 beat 时间
```

当前 BGM 提供 BPM 和拍型。

`BgmRecordPlayer` 把 BPM 同步给 `DrawSystem`。

`DrawSystem` 用 BPM 推导 `beatDuration`。

玩家输入移动、推箱、攻击或卡牌后，这些行为会先进入 `IntentSystem`。

`IntentSystem` 把玩家行动、敌人行动、刷怪行动整理成演出步骤。

`DrawSystem` 再按 `beatDuration` 播放角色、箱子、敌人和攻击反馈。

卡牌动画、结算面板、镜头 intro 等 UI 动画也通过 `BeatTiming` 跟随同一套速度。

`IntentSystem` 还会根据当前 BGM 的拍型切换敌人行动展示方式：

```text
4/4 或 2/4 风格：敌人行为可以用 AllInOneBatch 聚合到一个拍点里
3/4 或 6/8 风格：敌人行为会使用 AllInTwoBatch，保留更适合三拍或复合拍的 P / M / A 节奏
```

所以 BGM 的风格会反过来影响动作演出的速度和敌人行动分组。

项目里使用 Suno AI 生成了一组适合本玩法气质的 BGM。

它们不是只用来铺背景氛围，而是要给推箱子、出牌、敌人行动和结算反馈提供共同速度。

玩家可以在右上角 BGM 面板点击切歌。

切歌后，当前曲目的 BPM 和拍型会继续同步到动作演出中。

### 3D 编辑器与 RunConfig

3D 关卡编辑器用于快速生产和验证关卡空间。

RunConfig 用于配置开局、关卡循环、奖励、难度、商店、事件和 BGM。

两者配合后，策划可以完成：

```text
编辑关卡空间
放入 Classic 或 Escort 流程
调整奖励和压力
进入 Run 中验证体验
再回到编辑器继续调关
```

## 当前限制

```text
Classic 的压力切换还需要更多测试
Escort 的核心箱路径可读性仍需继续调关
悔棋代价曲线需要根据真实游玩数据调整
卡牌救局能力还需要更多内容覆盖
Skip 奖励需要避免过强或过弱
教程需要明确教会玩家 Classic 阶段、Escort 目标、悔棋和卡牌补救
```
