# 卡牌前端维护文档

日期：2026-05-09

## 目标

这份文档记录当前 `RunRoute` / `HandZone` / `CardView` 的卡牌前端方案，方便后续继续补卡、改交互、查 bug。

当前前端目标不是做完整杀戮尖塔复刻，而是先保证这几件事稳定：

- 手牌可读
- 拖拽出牌可用
- 待定瞄准可用
- 确认释放后能平滑回收
- 取消释放后能回到手牌
- 后续卡牌 UI 能继续扩展

---

## 现有结构

### `HandZone`

负责整套手牌终端逻辑：

- 抽牌
- 牌组 / 弃牌堆计数
- 手牌布局
- Hover / Drag / Pending / Confirm / Cancel
- 牌打出后的 Intent 路由
- 回收进弃牌堆
- 自动补牌

它是当前卡牌前端的核心控制器。

### `CardView`

负责单张卡的表现与输入入口：

- 卡面显示
- TMP 文本显示
- 鼠标进入 / 退出
- 左键点击
- 左键拖拽
- 拖拽结束
- 卡片展开 / 收起动画
- 卡牌本体位移 / 缩放 / 旋转 tween

### `CardSO`

负责卡牌静态数据：

- `cardId`
- 显示名
- 描述
- 图标
- 费用 / 数值
- 释放规则
- 其他设计配置

### `CardIntent`

负责运行时一次出牌的意图数据。

注意：现在传的是“卡牌实例意图”，不是只传 `cardId`。原因是卡牌数值、方向、目标、额外参数都可能在实例上变化。

## 当前交互链路

### 1. 手牌生成

```text
DrawPile -> CreateHandCard -> HandZone Relayout
```

新牌进入手牌后，先生成视图，再按照手牌槽位重新排布。

### 2. Hover

鼠标悬停时：

```text
CardView hover
  -> HandZone.HandleCardHoverEnter
  -> 卡片上半展开
  -> 位置抬高
```

鼠标移出时：

```text
CardView exit
  -> HandZone.HandleCardHoverExit
  -> 恢复手牌槽位
```

### 3. 拖拽出牌

```text
左键按住
  -> BeginDrag
  -> 卡牌跟随鼠标
  -> 松手离开 HandZone
  -> 进入 Pending
```

### 4. Pending 瞄准

Pending 状态表示：

- 牌已经拖离手牌
- 但还没确认打出
- 需要鼠标左键确认 / 右键取消

这一态下，卡牌应保持展开，不要提前收起。

### 5. 确认释放

```text
Pending -> 左键确认
  -> 解析释放目标
  -> 生成 CardIntent
  -> SetPlayerIntent
  -> TickSystem.PushTick
  -> 进入弃牌堆回收动画
```

确认释放时，卡牌允许在回收前收起，回收过程要保持平滑。

### 6. 取消释放

```text
Pending -> 右键取消
  -> 退回手牌
  -> 重排
```

取消不算出牌，只恢复到手牌态。

## 布局规则

### 手牌区域

- 手牌不要撑满整个屏幕
- 上下要留白
- 中轴线尽量居中
- 卡牌数量少时不要过度压缩
- 卡牌数量多时允许压缩到上限

### 手牌排序

- 默认是横向排布
- 当前采用锚点 / slot 计算，而不是纯 `HorizontalLayoutGroup`
- 手牌位置和旋转要统一由 `HandZone` 计算

### 卡牌尺寸

当前卡牌视觉上是“可伸缩上半部 + 固定下半部”的结构：

- 手牌态保持较短比例
- Hover 时上半展开
- 文本仍由下半部分承担

---

## 动画原则

### 1. 展开和收起都必须平滑

`CardView.SetHoverState(...)` 负责上半展开 / 收起 tween。

### 2. 位移和展开分离

卡牌位置、缩放、旋转由 `TweenTo(...)` 管。
展开区域由 `SetHoverState(...)` 管。

这两个动画不要耦死在一起，否则 Pending / Cancel / 回收时会互相污染。

### 3. 回收不要瞬切

确认出牌后：

- 先收起
- 再飞向弃牌堆

或者并行动画，但不能瞬间切状态。

---

## 输入与 Raycast

### 必要条件

卡牌必须有可命中的 UI Raycast 目标，否则：

- `IPointerEnterHandler` 不触发
- `IPointerExitHandler` 不触发
- 拖拽不会稳定

`CardView` 里已经做了兜底：

- 优先使用自身 `Graphic`
- 没有就自动补一个透明 `Image`

### 维护建议

- 不要把 `CardView` 做成 `Button`
- 直接用 `IPointer...` 系列
- 子节点如果接管 Raycast，要确认不会吞掉父节点事件

## 已知约束

- 当前还没有完整的卡牌效果系统 UI
- 当前卡牌更多是“前端框架”
- 释放规则还在不断补
- 初始卡组来自开局配置，后续可拆成更细的 SO
- 现在的重点是交互可靠，不是视觉最终稿

---

## 维护时的检查顺序

### 改卡牌表现时

先看这三个点：

1. `CardView`
2. `HandZone`
3. `CardSO`

### 改出牌逻辑时

先确认：

1. 当前卡是否还在 `_hand`
2. 是否处于 `_draggingCard`
3. 是否处于 `_pendingCard`
4. 释放规则是否能返回目标
5. `CardIntent` 是否装载了实例级数据

### 改布局时

先检查：

1. 手牌总宽度
2. 卡牌间距下限
3. hover 抬高距离
4. 上半展开高度
5. 弃牌堆 / 牌库位置是否仍可读

---

## 当前待跟进事项

- 手牌尺寸和展开比例仍可继续微调
- 后续新增更多卡牌后，需要补一份卡牌前端 prefab 规范
- 牌库界面、弃牌堆界面、抽牌动画可以继续扩展
