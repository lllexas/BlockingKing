# T13 3D 关卡编辑器维护指南

## 目标

3D 关卡编辑器用于在真实运行场景中编辑 `LevelData`，让编辑结果立刻通过现有 `LevelPlayer`、ECS 和 3D 地形显示链路反馈出来。

它不是第二套关卡运行场景，也不是轻量预览器。它复用当前场景里的 `GameFlowController`、`LevelPlayer`、`CameraController` 和运行时系统。

## 设计边界

入口在 `LevelData` 的 Inspector 上。

点击 `3D 编辑`时，Editor 侧只做三件事：

```text
GameFlowController.mode = LevelEdit
LevelPlayer.levelData = 当前 LevelData
Enter Play Mode
```

进入 Play Mode 后，由 `GameFlowController.StartLevelEdit()` 负责隐藏正式 UI、启动 `LevelPlayer`，并挂接 `Level3DEditorController`。

`Level3DEditorController` 只负责编辑交互、draft 数据、保存、放弃和即时重建预览。

## 核心数据源

编辑器有两个核心输入：

```text
LevelData
TileMappingConfig
```

`LevelData` 保存地图尺寸、地形 ID、tag 坐标。

`TileMappingConfig` 定义地形和 tag 的语义、颜色、编辑 tile、EntityBP 和 3D/ECS 解释方式。3D 编辑器的 palette 必须从 `TileMappingConfig.entries` 和 `TileMappingConfig.tagDefinitions` 生成。

不要只把 `LevelData` 当作完整数据源。没有正确的 `TileMappingConfig`，3D 编辑器无法保证 brush、ECS 实体和 3D 表现一致。

## 运行链路

当前调用链：

```text
LevelDataInspector.Open3DEditor
  -> 记录 GameFlow 原 mode、场景路径、Transform 索引路径
  -> 记录 LevelPlayer 原 levelData、场景路径、Transform 索引路径
  -> GameFlowController.mode = LevelEdit
  -> LevelPlayer.levelData = 当前 LevelData
  -> EnterPlaymode

GameFlowController.Start
  -> StartLevelEdit
  -> HideMainMenu / HideRunUiPanels / HideBgmRecord
  -> LevelPlayer.LoadConfiguredLevel
  -> Level3DEditorController.Configure

Level3DEditorController.ApplyBrush
  -> 修改 draft
  -> LevelPlayer.LoadLevel(draft, config)
  -> LevelPlayer.RebuildWorld
```

`LevelPlayer.RebuildWorld()` 会重新执行：

```text
BuildEntitiesInternal
CacheTargetCells
RebuildTerrainVisualsInternal
```

因此刷格后的反馈是整关重建，不是单格增量更新。当前阶段不追求增量性能。

## 保存与退出

Inspector 是编辑器主控入口。用户从 `LevelData` Inspector 进入，也从这里保存和放弃。

Play Mode 中，当前 `LevelData` Inspector 会显示：

```text
保存 3D 修改
放弃 3D 修改
退出 3D 编辑
```

保存调用 `Level3DEditorController.SaveChanges()`，把 draft 写回原始 `LevelData` 资产。

放弃调用 `DiscardChanges()`，丢弃 draft，重新从原始 `LevelData` 克隆并重建预览。

退出调用 `ExitEditor()`，也可以直接点击 Unity 顶部 Stop。退出 Play Mode 后，`LevelDataInspector` 的 tracker 会把 `GameFlowController.mode` 恢复到进入编辑器前的值。

恢复逻辑不要依赖 `GlobalObjectId`。Play Mode 退出后场景对象会重建，GlobalObjectId 在这个场景下不够稳。当前实现仿照 2D Tilemap 编辑器的持久化恢复思路，把可重建状态写入 `EditorPrefs`：

```text
GameFlowController scene path
GameFlowController Transform sibling index path
GameFlowController original mode
LevelPlayer scene path
LevelPlayer Transform sibling index path
LevelPlayer original LevelData GUID
```

回到 Edit Mode 后，tracker 会按 scene path 和 Transform index path 找回对象，并恢复：

```text
GameFlowController.mode
LevelPlayer.levelData
```

恢复完成后必须清理对应 `EditorPrefs`。如果这里失效，最明显症状是退出 Play Mode 后 `GameFlowController.mode` 仍残留为 `LevelEdit`。

## UI 维护要求

`Level3DEditorController` 使用 IMGUI 绘制编辑面板。由于 `TileMappingConfig` 中 tag 可以无限增长，面板必须固定区域，palette 必须滚动显示。

维护时遵守：

```text
不要让 tag/brush 无限撑高 OnGUI
不要让面板覆盖区域和鼠标拦截区域不一致
不要把保存/放弃主入口移到场景对象 Inspector
不要绕过 TileMappingConfig 生成 palette
```

当前面板区域分为：

```text
Toolbar
Cell Inspector
Filter
Palette ScrollView
Footer
```

palette 只能使用剩余高度，不能无限列出所有 tag。

## BGM 与正式 UI

`LevelEdit` 是编辑模式，不应显示正式 Run UI、主菜单 UI 或 BGM 唱片机 panel。

当前处理点：

```text
GameFlowController.StartLevelEdit
GameFlowController.HideRunUiPanels
BgmRecordAnimator.ShouldHideForFlow
BgmRecordOnGUIFrontend.OnGUI
```

如果后续新增正式 UI panel，应把它加入 `HideRunUiPanels()` 或对应的编辑模式隐藏条件。

## 维护警戒线

不要重新引入专用 3D 编辑场景。

不要创建第二套 `LevelPlayer`、`GameFlowController`、`CameraController`。

不要让 `QuickPlaySession` 参与 3D 编辑。`LevelEdit` 下 `LoadConfiguredLevel()` 必须使用当前 `LevelPlayer.levelData/tileConfig`。

不要把 `Level3DEditorController` 变成流程调度器。流程属于 `GameFlowController`，编辑交互属于 `Level3DEditorController`，入口与保存属于 `LevelDataInspector`。

不要用 `GlobalObjectId` 作为 3D 编辑退出恢复的唯一依据。恢复场景内组件时优先使用 scene path + Transform sibling index path，并保留必要的资产 GUID。

基础验收：

```text
点击 LevelData Inspector 的 3D 编辑可以进入 Play Mode
进入后 GameFlow 是 LevelEdit
主菜单、Run UI、BGM 唱片机不显示
palette 列出 TileMappingConfig 中的地形和 tag
palette 支持分类、搜索和滚动
刷地形/tag 后 ECS 和 3D 地形立即重建
保存会写回当前 LevelData
放弃会恢复到保存前的 LevelData
点击 Unity Stop 后 GameFlow mode 恢复原值
点击 Unity Stop 后 LevelPlayer.levelData 恢复原引用
```
