# T14 3D 关卡编辑器使用指南

## 用途

3D 关卡编辑器用于在 Play Mode 中编辑 `LevelData`，并用真实 `LevelPlayer`、ECS 和 3D 地形显示立即预览效果。

它适合检查 2D tile/tag 数据在 3D 运行环境中的真实表现。

## 进入编辑器

1. 在 Project 中选中要编辑的 `LevelData`。
2. 确认当前场景中已有 `GameFlowController` 和 `LevelPlayer`。
3. 确认 `LevelPlayer` 上的 `TileMappingConfig` 是当前项目要使用的配置。
4. 在 `LevelData` Inspector 中点击 `3D 编辑`。
5. Unity 会进入 Play Mode，并以 `GameFlowMode.LevelEdit` 启动。

进入后，主菜单、Run UI、BGM 唱片机面板会隐藏。

Play Mode 中可以直接在 Project/Inspector 选中另一个 `LevelData`，再点击它的 `3D 编辑` 切换编辑目标。

如果当前关卡有未保存修改，切换前会提示：

```text
保存并切换
放弃并切换
取消
```

## Palette

左侧 3D 编辑面板会从 `TileMappingConfig` 生成 palette。

分类包括：

```text
All
Terrain
Actors
Targets
Walls
Other
```

可以用搜索框按名称、ID、EntityBP 名称过滤。

palette 是滚动区域，tag 很多时不会无限撑开面板。

## 编辑操作

常用操作：

```text
鼠标左键        刷当前选中的 terrain/tag
Alt + 鼠标左键  选择当前格子的 tag；重复点击会在占位单位和 Target 之间循环
鼠标右键        优先擦除当前格子的 tags；没有 tags 时，只有选中 Terrain 类 brush 才会擦除 terrain
1-9             快速选择当前可见 palette 前 9 个 brush
Ctrl+Z          撤销
Ctrl+Y          重做
Ctrl+S          保存到 LevelData
Esc             退出 Play Mode
```

`Alt + 鼠标左键` 选择 tag 时，会优先选中当前格子的占位单位；如果同格还有 Target，再次 `Alt + 鼠标左键` 会切换到 Target。

选中后场景中会出现黄色选中框，屏幕右下角会显示 `Selected Tag` 面板，可查看默认/专有/最终使用参数，并编辑该 tag 实例的 `Override BP`。`创建专有 BP` 会从默认 BP 复制一个独立资产到当前 `LevelData` 同目录下：

```text
<LevelDataFolder>/<LevelName>_EntityBPOverrides/
```

鼠标悬停在格子上时，面板会显示当前 cell 的 terrain 和 tags。

每次刷格后会立即重建 `LevelPlayer` 世界：

```text
LevelPlayer.LoadLevel(draft, config)
LevelPlayer.RebuildWorld()
```

因此 ECS 实体和 3D 地形显示会立即刷新。

## 保存与放弃

保存和放弃的主入口在原 `LevelData` Inspector 上。

Play Mode 中，如果当前 Inspector 仍选中正在编辑的 `LevelData`，会显示：

```text
保存 3D 修改
放弃 3D 修改
退出 3D 编辑
```

`保存 3D 修改` 会把当前 draft 写回原始 `LevelData` 资产。

`放弃 3D 修改` 会丢弃 draft，并从原始 `LevelData` 重新加载预览。

`退出 3D 编辑` 会退出 Play Mode。也可以直接点击 Unity 顶部 Stop。

## 退出后的状态

退出 Play Mode 后，编辑器会自动恢复：

```text
GameFlowController.mode
LevelPlayer.levelData
```

如果退出后仍看到 `GameFlowController.mode = LevelEdit`，说明恢复流程失效，应检查 `LevelDataInspector` 中的 LevelEdit tracker。

## 注意事项

`TileMappingConfig` 是 3D 编辑器能否正确工作的关键配置。

如果 palette 缺内容、刷上去没有实体、3D 表现不对，优先检查：

```text
LevelPlayer.tileConfig 是否正确
TileMappingConfig.entries 是否完整
TileMappingConfig.tagDefinitions 是否完整
tagName 是否符合 LevelPlayer 识别规则
EntityBP 是否已配置
```

当前编辑器使用整关重建刷新，不做单格增量更新。大关卡可能有短暂卡顿，这是当前可接受行为。
