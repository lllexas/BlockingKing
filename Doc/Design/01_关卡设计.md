# 关卡设计纲要

## 核心体验目标

```
策略记忆 → "大概知道怎么走" → 不要求极端背板
单关时长 → 正常游玩 3~4 分钟（玩家可以思考更久）
张弛有度 → 不是每个地方都是高压锅
```

---

## 关卡类型

### 类型 A：经典关卡（奖励关）

```
核心：使用经典推箱子的高难关卡（Microban / Original 精选）。
     没有怪物。大量箱子，大量目标点。
定位：Run 中的放松环节。纯正激励。

步数 = 最优解 × 1.2
  举例：本关最优解需 50 步 → 给你 60 步
  不是固定的 100 步，而是跟着关卡本身走

同一关，三种人三种爽法：
  老ass（不出牌）：基础移动全收箱子 → "我技术真牛"
  普通玩家（牌一般）：收一半 → "还行，拿了不少奖励"
  构筑成型（牌太好）：破墙、跳日、冲车暴力全收 → "我这牌组真猛"

  经典关卡不是用 WFC 生成的。
  它是真材实料的高难推箱子谜题。
  WFC 只用于带球关卡。

正激励设计：
  推 1~2 个箱子到目标 → 1 张随机普通卡
  推 3~4 个箱子到目标 → 2 张卡（至少 1 张稀有）
  推 5~7 个箱子到目标 → 2 张卡 + 1 个道具
  推 8+ 个箱子到目标 → 卡牌升级（选一张卡升阶）

  "推不到也不惩罚"
  推少了只是拿得少，不是失败
```

### 类型 B：带球关卡（护送关）

```
核心：1~2 个箱子（球），从地图一端护送到另一端。
定位：Run 中的高压环节。怪物持续骚扰。

规则：
  球箱必须存活到达目标区域
  球箱被打爆 → 本关失败
  球箱 HP 在 Run 内继承（跨关卡持久化）
  玩家与球箱共享血量

共享血量的含义：
  怪物攻击球箱 → HP 扣减 → 玩家也受伤
  怪物攻击玩家 → HP 扣减 → 球箱也受伤
  本质是一体两面：你死了球也没人护了

护送路径：
  起点区 → 中场 → 终点区（目标格）
  怪物从地图边缘生成，向球箱移动
  玩家需要一边推球一边打怪/卡位
```

## 张弛节奏

```
经典关卡 = 松（放松推箱子，刷奖励）
带球关卡 = 紧（高压护送，生死一线）

一个 Run 的理想节奏：

  带球 1 → 经典 1 → 带球 2 → 经典 2 → 商店 → 
  带球 3 → 事件 → 经典 3 → 带球 4(BOSS)

  紧 → 松 → 紧 → 松 → 休 → 紧 → ? → 松 → 极紧
```

经典关卡穿插在带球关卡之间，起到"喘息"作用。玩家在高压护送后回到纯粹推箱子，不仅能恢复状态，还能用经典关卡刷到的奖励反哺后面的带球关卡。

## 地图来源

### 经典关卡：手选高难推箱子关卡

```
来源：Microban / Original / Sasquatch 等经典关卡库
不是 WFC 生成，是真实的、经过验证的推箱子谜题

每关预先计算最优解步数
  → 可用 Python 求解器暴力搜出最优解
  → 步数预算 = 最优解 × 1.2

这些关卡本身就是"精妙的"，不需要烂。
经典关卡的价值是让玩家享受纯粹的推箱子乐趣。
```

### 带球关卡：WFC 坍塌生成

```
来源：WFC 算法实时生成

生成约束：
  1~2 个球箱（带特殊标记）
  起点区在一侧，目标区在对侧
  中间有通道、障碍物
  怪物刷新点在地图边缘

WFC 的"狗屁不通"在带球关是核心体验：
  通道卡地形 → 交牌穿过去，但怪物在追
  球箱卡死角 → 双倍焦虑，球在掉血
  不懂怎么走？→ 策略记忆 — 大概知道方向就好
```

## 带球关卡 PCG 参数设计原理

### 两大元数据

```
带球关卡 = f(曼哈顿距离, 曼哈顿斜率)

  曼哈顿距离  = Δx + Δy          → 长度难度系数
  曼哈顿斜率  = Δy / Δx (原值)   → 卡牌价值权重
```

### 长度难度：曼哈顿距离

```
距离越长 → 回合越多 → 怪物行动次数越多 → 球箱受到的威胁越大

难度随距离线性增长：
  短途（8~12 格）：2~3 只怪物，教学/热身
  中途（14~18 格）：3~5 只怪物，标准运营
  长途（20~26 格）：5~7 只怪物，高难/BOSS

距离直接决定"你有多少回合会被打"。
```

### 斜率难度：卡牌价值的关卡绑定

核心观点：**同一个起点，不同的斜率，让不同卡牌成为本关的 MVP。**

```
实际斜率 k = Δy / Δx

使用对数斜率：log_slope = lg(k)

  k = 1   (45°)  → log_slope = 0        ← 基准线
  k → 0   (水平) → log_slope → -∞
  k → ∞   (垂直) → log_slope → +∞

取绝对值：|log_slope| = 与 45° 的偏差
```

**为什么用对数？**

```
水平（k≈0）和垂直（k≈∞）在绝对值上不对称，
但对数让它们对称：
  |lg(0.01)| ≈ 2  ≈  |lg(100)|

水平路径和垂直路径，对斜向卡牌来说一样废。
对数表达了这种"等价"。
```

**斜率如何影响卡牌效率**

```
令 k = Δy/Δx (实际斜率)
令 x = ln(k)   (对数斜率)

主教系（斜向）效率：
  eff_B(k) = (1+k) / (√2 × √(1+k²))

城堡系（直线）效率：
  eff_R(k) = √(1+k²) / (1+k)

乘积恒为常数：
  eff_B × eff_R = 1/√2 ≈ 0.707   （不随 k 变化！）
```

**各点数值：**

| x = ln(k) | k | eff_B | eff_R | eff_B + eff_R | 谁值班 |
|-----------|----|-------|-------|---------------|--------|
| 0 | 1 | 1.000 | 0.707 | **1.707** | 主教当爹 |
| ±0.5 | 0.607 / 1.649 | 0.963 | 0.734 | 1.697 | 主教优 |
| **±0.66** | **0.217 / 4.61** | **0.841** | **0.841** | **1.682 (最低)** | **骑士值班** |
| ±1.0 | 0.368 / 2.718 | 0.909 | 0.778 | 1.687 | 骑士优 |
| ±2.0 | 0.135 / 7.39 | 0.796 | 0.888 | 1.684 | 城堡优 |
| ±∞ | 0 / ∞ | 0.707 | 1.000 | 1.707 | 城堡当爹 |

```
和值曲线（对称）：

 1.707 ┤●                           ●
       │  ╲                       ╱
 1.695 ┤    ╲                   ╱
       │      ╲    ╱───╲     ╱
 1.682 ┤        ╲●╱     ╲●╱      ← 最小值点 (骑士值班)
       │
       └──┬──────┬──────┬──────┬──→ |ln(k)|
          0     0.5    0.66    1
```

**三个区域：**

```
|ln(k)| ≈ 0       → eff_B = 1.0, eff_R = 0.707  → 主教当爹
|ln(k)| ≈ 0.66    → eff_B = eff_R = 0.841       → 两人都不顶用，骑士值班
|ln(k)| → ∞       → eff_B = 0.707, eff_R = 1.0  → 城堡当爹

和值在两端与中心都是 1.707，谷底在 |ln(k)| ≈ 0.66。
这意味着：
  - 大部分斜率下，总有一系牌勉强够用
  - 但在 |ln(k)| ≈ 0.66 时，两系同时疲软
  - 这个区间 = 骑士系的"专属舞台"
```

**斜率 = 隐形的关卡特征**

```
玩家不知道下关的斜率。
所以不能堆单一类型的牌。

  拿了 3 张主教系 → 遇到水平关 → 牌组废了一半
  拿了 3 张城堡系 → 遇到 45° 关 → 效率还行但缺灵活性

斜率这个参数，天然逼你构筑多样化。
```

### 怪物密度随距离自动缩放

```
怪物数量 N = floor(曼哈顿距离 / 4) + 难度修正

  难度修正：
    普通关：  -1
    标准关：   0
    高难关：  +1
    BOSS关：  +3

  举例：
    距离 12、普通关 → N = 3 - 1 = 2
    距离 20、BOSS关 → N = 5 + 3 = 8
```

### WFC 输入元数据

```
每个带球关卡只传两个参数给 WFC：

{
    manhattan_distance: 16,      // 曼哈顿距离
    log_slope:    0.3,           // |lg(Δy/Δx)|，与 45° 的偏差
    ball_count:   1,             // 球箱数量
    difficulty:   "normal"       // 难度档位 → 缩放怪物数
}

WFC 不关心"答案"。它只需要知道起点和终点。
烂地图靠卡牌运营走通。
```

## 地图生成策略：先钉城市，再修路

### 核心问题

WFC 直接在全图上撒 tile 会产出"无敌破坏王"——满地碎块、死路、孤立区域，看着不像推箱子。

推箱子需要**空间感**：开阔的"房间" + 狭窄的"通道"。不是满屏碎砖。

### 城市—道路两步法

```
Step 1: 钉城市（放置房间）

  在地图上放置若干个矩形"房间"：
    房间 = 开放空间，用于容纳箱子、目标点、怪物刷新点
    房间之间故意错位，形成走廊的"弯曲"
    房间数量随曼哈顿距离增长（距离越长，中继房间越多）

Step 2: 修路（连通城市）

  对每个房间，检查与邻近房间的连通性：
    能水平直连？（两房间 y 范围有重叠，且之间无墙）
      → 连水平走廊
    能竖直直连？（两房间 x 范围有重叠，且之间无墙）
      → 连竖直走廊
    两端都能连？
      → 选一端连（避免环）

  走廊宽度 = 1~2 格
  走廊方向 = 纯水平或纯竖直（不产生斜走廊）

  起点 → 城市1 → 城市2 → ... → 终点
  整条路径就是护送球箱的"干线"
```

**连通规则图解：**

```
  [房间A]          [房间A]
     │ 竖直走廊        ╲
  [房间B]          [房间B]  水平走廊
  
  ✓ 水平/竖直可连才连      ✗ 斜向不连，需要中继房间
```



### 为什么这样就像推箱子

```
推箱子的经典地图结构：
  不是"满地砖块"，而是"几个房间 + 几条走廊"

  房间 = 操作空间（推箱子、打怪）
  走廊 = 移动空间（把球从一个房间推到下一个）
  
  "烂"发生在房间里：
    → 箱子堵住门
    → 目标点在墙后
    → 怪物从角落刷出来

  走廊保证路径存在 —— 你不会无路可走
  房间保证狗屁不通 —— 你仍然需要交牌
```

### WFC 职责重新划分

```
WFC 不再是"生成整个地图"
WFC 是"在城市之间修路 + 在城市内摆障碍"

  城市布局（矩形位置、大小）→ 由预设模板或简单算法决定
  走廊连通               → WFC 坍塌
  房间内部（箱子、障碍）   → WFC 坍塌（狗屁不通的部分）
  怪物刷新点             → 房间角落随机放置
```

这保证了地图"看起来像推箱子"，但"走起来像肉鸽"。

## 房间内部：球箱 + 奖励箱的双层设计

### 带球关卡不只是"护送"

```
只护送 → 就一个箱子从起点推到终点 → 中间只是赶路 + 打怪

但推箱子的乐趣在"多箱子之间的阻隔和腾挪"。

解法：在每个房间里撒奖励箱。
```

### 奖励箱设计

```
球箱：   必须从起点护送到终点，被打爆 = 失败
奖励箱： 房间内的可选目标
         推到目标点上 → 当场领取奖励（卡/道具/回血）
         推不到 → 不惩罚
         可以放弃（不挡路就行）

奖励箱是"引诱"不是"强迫"。
它制造了经典推箱子的多箱阻隔体验，
但你可以选择无视它继续护送球箱。
```

### 一个房间的内部结构

```
┌─────────────────────┐
│  🎁  ·  🧱  🧱     │  ← 奖励箱 + 目标点，可选
│         🎯          │
│  🧱  🧱     🎁     │
│           🎯        │
│  ═══ 入口          │
│             ║       │  ← 出口到走廊
│  🧱  🎁  🧱  🎯   │
│  👾                 │  ← 怪物从角落刷
└─────────────────────┘

🎁 = 奖励箱   🎯 = 目标点   👾 = 怪物   🧱 = 障碍

主路径：入口 → 推着球箱穿过 → 出口
可选：停下来推奖励箱到目标 → 拿奖励
风险：停下来 = 怪物逼近球箱
```

### 奖励箱 + 卡牌 = 经典推箱子微缩版

```
奖励箱被墙围住？→ 破墙拿
奖励箱和目标点在斜对角？→ 主教行
两个奖励箱互相堵路？→ 骑士跳绕后
全部放弃只推球箱？→ 也可以，但你亏了奖励
```

这就是在护送中嵌入的"经典推箱子微关卡"。不是必须解，但你手上有牌就想试试。

### 关于房间地形

```
基础：矩形房间
进阶：对经典推箱子地图做统计分析，提取 tile 分布规律来训练 WFC

先做矩形。矩形已经能提供足够的变化（大小、长宽比、内部障碍）。WFC 从经典地图学出来的地形是加分项，不是必需品。
```

## 怪物刷新机制

```
带球关卡开始 → 怪物在地图边缘生成
怪物生成位置：距离球箱至少 5 格
怪物行为：每回合向球箱走 1 格，贴脸后攻击
怪物不会去管经典关卡里的普通箱子（只有球箱是目标）

怪物数量 = 本关难度参数
球箱HP = Run内继承（比如开局 20 HP）
```

---

## 经典关卡奖励机制

```
推 1~2 个箱子到目标 → 获得 1 张随机普通卡
推 3~4 个箱子到目标 → 获得 2 张卡（至少 1 张稀有）
推 5~7 个箱子到目标 → 获得 2 张卡 + 1 个道具
推 8+ 个箱子到目标  → 获得 卡牌升级（选一张卡升阶）
```

不做步数惩罚。不因为步数多用完就惩罚玩家。100 步是"尽情推"的时间，不是倒计时。玩家在经典关卡里是**被奖励的**，不是**被审判的**。
