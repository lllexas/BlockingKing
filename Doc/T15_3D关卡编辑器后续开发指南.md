# T15 3D 关卡编辑器后续开发指南

## 当前架构

3D 编辑器由三部分组成：

```text
LevelDataInspector        入口、保存、放弃、退出、恢复临时场景设置
GameFlowController        LevelEdit 启动流程、隐藏正式 UI、启动 LevelPlayer
Level3DEditorController   Play Mode 下的编辑交互、draft、palette、重建预览
```

不要新增独立 3D 编辑场景。

不要复制 `LevelPlayer`、`GameFlowController`、`CameraController`。

## 数据原则

3D 编辑器的真实输入是：

```text
LevelData + TileMappingConfig
```

后续功能必须围绕 `TileMappingConfig` 扩展，而不是硬编码 tag ID。

可接受的 fallback 只应放在 `LevelPlayer` 这种运行时兼容层，不应成为编辑器 palette 的主要来源。

## 可扩展方向

### 1. Brush 元数据

现在 brush 从 `TileMappingConfig.entries/tagDefinitions` 直接生成。

后续可以给 `TileMappingConfig.TagDefinition` 增加编辑器元数据，例如：

```text
category
displayName
description
icon
placementRule
exclusiveGroup
```

这样可以替代当前按 tagName 前缀推断 Actor/Target/Wall 的逻辑。

### 2. 放置规则

当前冲突规则很简单：

```text
Target 类 tag 互斥
非 Target 占格类 tag 互斥
```

后续应把规则显式数据化，例如：

```text
同格只能有一个 Actor
同格只能有一个 Target
Wall 不能和 Actor 同格
Player 必须唯一
CoreBox 必须唯一或按模式约束
Target.Enemy 必须有 Spawn 配置
```

规则应该在刷入前校验，并在 UI 中给出原因。

### 3. 验证面板

建议增加关卡验证区，检查：

```text
是否有 Player
是否有 Box
是否有 Target
Box/Target 数量是否符合玩法模式
tag 是否越界
tagID 是否不存在于 TileMappingConfig
EntityBP 是否缺失
地图是否存在不可达区域
```

验证结果应显示在 `LevelData` Inspector 或 3D 编辑面板中。

### 4. 更细的编辑模式

当前只有一个刷子模式。

后续可以拆成：

```text
Paint Terrain
Paint Tag
Erase Tag Only
Erase Terrain Only
Inspect
Move Actor
Rectangle Fill
Flood Fill
```

不要在没有明确输入规则时直接扩展鼠标行为，避免和相机控制冲突。

### 5. 保存策略

当前保存是手动保存 draft 到 `LevelData`。

后续可以增加：

```text
Auto-save draft backup
Dirty confirmation before Exit
Save As duplicated LevelData
Change summary
```

但默认仍应保持“退出不自动保存”，避免误写资产。

## UI 开发要求

当前 UI 使用 IMGUI。

维护时必须保证：

```text
面板固定在屏幕内
palette 必须滚动
tag 数量无限时 UI 不撑爆
鼠标在面板上时不刷地图
按钮高度固定
长文本不影响整体布局
```

如果迁移到正式 UI 系统，仍要保留 `LevelData` Inspector 上的保存、放弃、退出主入口。

## 恢复机制

进入 3D 编辑前，`LevelDataInspector` 会记录：

```text
GameFlowController scene path
GameFlowController Transform sibling index path
GameFlowController original mode
LevelPlayer scene path
LevelPlayer Transform sibling index path
LevelPlayer original LevelData GUID
```

退出 Play Mode 后按这些信息恢复。

不要把恢复逻辑改回只依赖 `GlobalObjectId`。Play Mode 退出后场景对象可能重建，GlobalObjectId 在这里不够稳。

## 不要做的事

不要让 `QuickPlaySession` 参与 3D 编辑。

不要让 `Level3DEditorController` 负责 GameFlow 路由。

不要在 3D 编辑器中启动正式 Run 流程。

不要为了临时 UI 创建第二套场景对象。

不要绕过 `TileMappingConfig` 直接写固定 tag ID。

## 推荐验收

每次改动后至少检查：

```text
进入 3D 编辑后 GameFlow 为 LevelEdit
退出后 GameFlow 恢复原 mode
退出后 LevelPlayer.levelData 恢复原引用
BGM 唱片机和正式 UI 不显示
palette 能列出 TileMappingConfig 内容
刷地形后 3D 地形刷新
刷 tag 后 ECS 实体刷新
保存写回 LevelData
放弃不写回 LevelData
```

```powershell
dotnet build Assembly-CSharp.csproj -v minimal
```
