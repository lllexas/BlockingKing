# CNN 地形生成 — 训练记录

## 日期
2026-05-07 ~ 2026-05-08

---

## 目标
训练小型 CNN，输入 7×7 tile 窗口，输出中心格子的 tile 概率分布。
用于 WFC 带球关卡的房间内部地形填充。

## 输出
5 类地形：墙 / 地板 / 目标 / 箱子 / 箱子在目标
（玩家 `@` 并入地板，人在目标 `+` 并入目标）

---

## 最终架构

```
7×7 → Conv1(3×3, 1→8)  → 5×5×8   ReLU
    → Conv2(3×3, 8→16) → 3×3×16  ReLU
    → Conv3(3×3, 16→16)→ 1×1×16  ReLU
    → FC(16→5)
```

- 感受野：7×7（覆盖全窗口）
- 参数量：3653（<5000 约束）
- 推理：纯 C# CPU <1ms

### 为什么用三层卷积
- 第 1 层：学墙-地板边界
- 第 2 层：学转角、T 字路口
- 第 3 层：学围合空间结构（房间 vs 走廊）

### 为什么不用更大的模型
- 3653 参数已足够
- Unity 端纯 C# 推理，参数越多越慢
- 此任务本质是局部 pattern 匹配，不是全局理解

## 产出的权重

```
weights/
├── epoch01/   ← 最软：Val Loss 0.1014，概率多样
├── epoch02/   ← 偏软：Val Loss 0.0001，开始收敛
├── epoch03/   ← 偏硬：Val Loss 0.0000
└── epoch05/   ← 最硬：Val Loss 0.0000，铁口直断
```

每组权重包含 8 个 .bin 文件：
- `conv1_weight.bin` / `conv1_bias.bin` — [3,3,1,8] / [8]
- `conv2_weight.bin` / `conv2_bias.bin` — [3,3,8,16] / [16]
- `conv3_weight.bin` / `conv3_bias.bin` — [3,3,16,16] / [16]
- `dense_weight.bin` / `dense_bias.bin` — [16,5] / [5]

每组附带 `diary.txt` 记录生成时间、架构、训练指标。

---

## 使用建议

1. 取 4 组权重分别接入 Unity C# 推理代码，跑 WFC 生成
2. 对比每组生成的地图——epoch 越早，概率越"软"，WFC 采样越多样
3. 选效果最好的一组用，其他删掉
4. 注意：`@` 和 `+` 被合并到地板和目标中，WFC 不需要生成玩家位置
