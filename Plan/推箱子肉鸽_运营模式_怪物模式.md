# 推箱子肉鸽 — 运营模式（怪物模式）

## 与解密模式的关系

| | 解密模式 | 运营模式 |
|---|---|---|
| 地图 | 精心设计 | WFC 坍塌生成（故意烂） |
| 压力 | 步数 | 怪物打箱子 |
| 玩法 | 找最优解 | 运营卡牌资源 |
| 节奏 | 慢，可 SL | 快，一眼过 |
| 卡牌 | 锦上添花 | 生存必需品 |
| 核心体验 | 烧脑 | 肌肉记忆 |

两个模式共用同一套卡牌系统，共存在一个 Run 里，互相调剂。

---

## 核心设计

### 地图：WFC 坍塌生成

```
不用精心设计关卡。
WFC 自动生成，必然出现"狗屁不通"的地方：
  ├── 箱子卡在角落
  ├── 目标点被墙困住
  ├── 箱子离目标十万八千里
  ├── 通道太窄绕不过去
  └── 箱子堵住了唯一的通路

没关系。
肉鸽就是干这个的——把狗屁不通的地方走通。
你的卡牌就是"走通"的手段。
```

### 怪物：回合制，每一步都会动

```
每回合（玩家的每一步）：
  玩家操作：
    ├── 基础移动/推箱子 → 推进谜题进度
    └── 出牌 → 打怪 / 位移 / 破墙
    ↓
  全场怪物各自行动一格：
    ├── 向最近的箱子移动
    ├── 贴到箱子 → 攻击 → 箱子扣血
    └── 箱子 HP 归零 → 箱子碎了 → 本关目标数无法达成 → 失败
```

### 卡牌：同一套牌，两种用途

```
手牌可以用于：
  ├── 破谜题（位移、破墙、交换等）→ 推进推箱子进度
  └── 打怪（攻击牌、控制牌）      → 保护箱子不被干碎

但每回合只能做一件事。
用牌加速推箱子 → 怪物继续逼近
用牌打怪保箱子 → 推箱子进度归零

同一套牌，在"进度"和"存活"之间分配。
这就是运营的核心决策。
```

### 卡牌体系（共用 + 新增）

#### 解密系（与国际象棋位移共用）

| 卡牌 | 效果 | 运营模式下的价值 |
|------|------|-----------------|
| **骑士跳** | 走日，无视路径 | 跨过怪物群快速接近箱子 |
| **车行** | 直线滑动到撞墙 | 跨越地形快速转移 |
| **主教行** | 斜线滑动到撞墙 | 斜向绕开怪物 |
| **冲车** | 推整排箱子 | 暴力清路 |
| **贯穿** | 穿过箱子 | 被怪物挡住时穿过去 |
| **皇后驾到** | 任意方向到撞墙 | 终极机动 |

#### 怪物系（新增）

| 卡牌 | 效果 | 稀有度 |
|------|------|--------|
| **挥击** | 打周围 4 格一个怪物 N 点伤害 | 基础 |
| **横扫** | 打周围 8 格全部怪物 N 点伤害 | 普通 |
| **猎人狙击** | 指定任意位置的怪物，远程攻击 | 稀有 |
| **冰霜陷阱** | 放一个陷阱，下回合踩到的怪物冻住 3 回合 | 稀有 |
| **推怪** | 把怪物推 1 格（可推撞墙扣血） | 普通 |
| **斩首** | 秒杀一个 HP < 50% 的怪物 | 史诗 |
| **恐惧怒吼** | 周围怪物退后 2 格 | 普通 |

#### 双用途（新概念）

| 卡牌 | 效果 |
|------|------|
| **冲击波** | 对目标方向推箱子 2 格，路径上的怪物被推开并受伤 |
| **换位打击** | 与怪物互换位置，怪物眩晕 1 回合 |

## 怪物设计

```
怪物是"移动的压力源"，不是"要你打的怪"。

你不需要清怪来通关。
你需要保护箱子不被怪干碎。
你可以选择打怪，也可以选择不理它快速推完箱子。
```

### 怪物类型

| 怪物 | 行为 | 威胁 |
|------|------|------|
| 🟢 **小怪** | 每回合向最近箱子走 1 格，贴脸后攻击（1 点/回合） | 低 |
| 🟡 **冲锋怪** | 每回合走 2 格，但只攻击离它最近的箱子 | 中 |
| 🟠 **破箱者** | 每回合走 1 格，攻击力 3 点 | 高 |
| 🔴 **幽灵** | 穿墙走 1 格，无视地形 | 极高 |
| 🟣 **分裂怪** | 被击杀后分裂成 2 个小怪 | 特殊 |

### 怪物生成

```
每关开始，WFC 生成地图后，在地图上随机放置怪物。
怪物数量 = 关卡难度参数
怪物的初始位置不会紧贴箱子（至少隔 3 格）
```

## 关卡流程

```
[开始一关]
    ↓
[WFC 生成地图] → 必然有"狗屁不通"的地方
    ↓
[放置箱子 + 目标点 + 怪物]
    ↓
[玩家回合开始]
    ↓
┌───────────────────────────────┐
│  每回合：                      │
│    1. 玩家选择一个动作         │
│        基础移动/推箱子         │
│        或打出卡牌             │
│    2. 全场怪物各自行动 1 格    │
│    3. 怪物攻击范围内箱子        │
│    4. 检查：所有箱子上目标？    │
│         └── 是 → 通关          │
│    5. 检查：箱子HP归零？        │
│         └── 是 → 箱子碎了       │
│    6. 检查：剩余箱子数 < 目标数？│
│         └── 是 → 本关失败       │
└───────────────────────────────┘
```

## 教学：第一关就教玩家把墙打破

```
不用教学关卡。不需要。
第一关的 WFC 地图烂到：
  - 箱子卡在死角里，常规推箱子推不出来
  - 怪物已经走过来了

此时你手上有：【破墙】【骑士跳】

你不交牌 = 箱子被拆 = 失败
你交牌了 → 破墙/跳走 → 活了 → 你学会了

地图自己就是教学。
"打破规则"就是这游戏的规则。
```

## Run 内模式混搭

```
一个完整的 Run，12 关：

  第1关  [运营] 入门 — 学会破墙
  第2关  [解密] 经典推箱子 — 放松，攒牌
  第3关  [运营] 怪物 + 烂地图
  第4关  [解密] 精心设计的经典关卡
  第5关  [运营] 怪物数量增多
  第6关  [事件] 商店 / 休息
  第7关  [运营] 分裂怪出现
  第8关  [解密] 高难度手搓关卡
  第9关  [运营] 幽灵出现
  第10关 [解密] 最后的解密
  第11关 [运营] BOSS — 超多怪物
  第12关 [解密] 最终谜题
```

两种模式互相调剂：解密累了打打怪，打怪累了推推箱子。

## 与解密模式共享什么

| 共享 | 内容 |
|------|------|
| **卡牌系统** | 同一套牌库、抽牌洗牌机制 |
| **地图数据** | LevelData 的 int[][] 格式（WFC 也输出这个格式） |
| **网格渲染** | GridManager 共用 |
| **Run 结构** | 集成战略式路线选择 |
| **道具系统** | 药水位一次性消耗品 |
| **牌库构筑** | 过关选牌/删牌/升级 |
