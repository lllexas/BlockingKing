# 2026-05-14 Playtest 前打磨路线

当前目标：不公开 Steam 商店页，先准备隐藏 Playtest / 私发 key 所需的稳定测试版本。

## 当前判断

项目已经不是纸面规则或 PPT demo：

- 一局 Run 可以玩 30 到 60 分钟。
- Classic / Escort / Skip / Shop / Event / Settlement 主流程已经存在。
- 7 张核心卡、Escort 生成、围棋连子 Aura、RunConfig、3D 关卡编辑器和 UI 骨架已经落地。

当前最大风险不是内容系统是否存在，而是：

- 首局太难。
- 新玩家不知道为什么难。
- 数值、关卡池、刷怪、奖励还没经过足够试玩收束。
- 教程和可读性还没有完全承担陌生玩家入口。

## P0：先让陌生玩家玩得懂

1. 新手 Run / 首局配置
   - 前 10 到 15 分钟降低敌人压力。
   - 提高容错：金币、回血、悔棋、Skip 奖励可以偏宽。
   - 避免第一局抽到高复杂度 Classic 或 Escort。

2. 教程入口
   - 参考 `Reference/Tutorial/教程分镜_六幕方案.md`。
   - 第一目标不是讲完所有规则，而是让玩家经历一次“推错了也能救回来”。

3. 失败可解释
   - 玩家死亡、核心箱被毁、敌人刷出、目标未完成都要有清楚反馈。
   - 第一次失败后，玩家应该知道下一局要改什么。

## P1：调顺 30-60 分钟 Run

1. 难度曲线
   - 按 `Reference/RunConfig/肉鸽难度曲线与Round配置_操作指南.md` 调。
   - 优先改配置资产，不先改代码。
   - 每次只改少量变量，试玩后记录体感。

2. 内容密度
   - 不急着扩到完整肉鸽内容量。
   - 先保证现有卡牌、围棋兵、Classic、Escort 的节奏差异明显。

3. 结算与奖励
   - Classic 应该像喘息和奖励。
   - Escort 应该像压力和推进。
   - Skip 不应该像惩罚，而应该像低收益保底。

## P2：Playtest 包装

1. Build 稳定性
   - 从 Windows build 启动，不依赖 Unity Editor。
   - 跑完至少一局。
   - 检查音频、shader、UI、存档/配置初始态。

2. 反馈收集
   - 使用 `2026-05-14_试玩反馈记录.md` 记录每局。
   - 重点记录“难”的来源：规则、可读性、数值、关卡、卡牌、UI。

3. Steam Playtest
   - 暂不公开商店页。
   - 使用隐藏 Playtest + key 私发。
   - KYC / 税务 / 银行审核通过后继续创建 app 和 playtest 配置。

## 本阶段不优先做

- 大量新敌人。
- 大量新卡。
- Boss。
- 完整遗物系统。
- 公开商店页宣传。
