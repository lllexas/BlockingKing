# 2026-05-13 新增需求优先级

日期：2026-05-13

这页记录当前新增需求。P0 优先进入施工队列；P1 做体验补强；P2 暂存为后续优化项。

## P0：核心体验必须补齐

### 1. 教程

目标：
- 建立新玩家的第一轮理解路径。
- 解释基础移动、推箱、敌人意图、卡牌释放、胜负条件。
- 教程流程要能进入正式 run，也要能跳过。

关注点：
- 教程不能依赖口头说明，必须由关卡、UI 和交互反馈共同完成。
- 教程节奏要短，优先保证玩家能独立完成第一关。
- 需要避免和正式关卡生成、run 结算、牌库状态互相污染。

### 2. 牌库和商店面板 SpaceUIAnimator 化

目标：
- 牌库面板、商店面板接入正式 SpaceUIAnimator 体系。
- 显隐、进入、退出和按钮状态要和现有 run UI 保持一致。

关注点：
- 不再依赖临时 OnGUI 或直接 SetActive 作为主要表现。
- 面板显隐不能打断卡牌奖励动画、商店购买流程或 run round 背景板。
- 需要明确面板 UIID、入口按钮、返回逻辑和互斥关系。

### 3. 爆金币动画优化

目标：
- 获得金币时有明确、可读、带节奏的视觉反馈。
- 金币来源、飞向、落账时机要和 UI 数值变化对应。

关注点：
- 奖励结算、escort 推箱奖励、战斗结算、商店消费后的金币变化都要口径一致。
- 动画不能只服务单一场景，最好走统一金币变化表现入口。
- 需要避免面板关闭、场景状态切换时看不到动画。

## P1：强体验优化

### 1. 左键代理释放卡牌时创建新的卡牌半弹出状态

目标：
- 左键代理释放卡牌时，卡牌进入一个明确的半弹出/待确认状态。
- 玩家能看出当前正在选择或代理释放某张牌。

关注点：
- 半弹出状态要和 hover、drag、selected、pending release 区分清楚。
- 退出条件要明确：确认释放、取消、切换卡牌、进入锁定状态。
- 不应破坏现有手牌布局和奖励卡牌动画。

### 2. 优化攻击范围预览

目标：
- 卡牌或单位攻击范围预览更清楚，能表达有效目标、无效目标和实际命中路径。

关注点：
- 区分“可选择格”“实际命中格”“被阻挡路径”“无效释放”。
- 需要兼容直线、斜线、骑士跳、范围踩踏、冲锋等不同释放规则。
- 左键代理释放卡牌时，预览要和半弹出状态联动。

## P2：后续优化

### 1. 不期而遇面板优化

目标：
- 提升不期而遇面板的信息组织、选择反馈和进入/退出表现。

关注点：
- 和 run round 背景板共存，不要进入事件时关闭底板。
- 事件选项、奖励、代价、结果反馈要更明确。
- 后续可接入正式 SpaceUIAnimator。

### 2. 炮兵逻辑优化

目标：
- 优化炮兵行为逻辑，使其更稳定、更可读、更符合战术定位。

关注点：
- 检查目标选择、攻击范围、阻挡、墙体互动和意图展示。
- 炮兵行为需要让玩家能提前判断，而不是只靠试错理解。
- 优化前先明确当前炮兵在敌人体系里的职责。
