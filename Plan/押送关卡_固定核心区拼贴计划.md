# 押送关卡固定核心区拼贴计划

## 目标

押送关卡不再从素材地图里随机抽取一个普通箱子升格为 `Box.Core`。

生成流程改为先确定押送玩法的两个核心设施：

```text
Box.Core 起点区
Target.Core 终点区
```

然后再把旧关卡素材作为中间城市拼贴到地图上。

这套方案的核心原则是：核心玩法锚点必须是确定性的，素材拼贴只负责提供地图变化，不负责决定押送目标。

## 当前问题

当前实现会先根据 route 参数计算一个锚点，再用模板里的随机可走格去对齐城市，最后还会做坐标归一化。

这虽然可以运行，但逻辑上不够干净：

- 核心点的位置会受到素材地图形状影响。
- `Box.Core` 曾经依赖普通箱子的随机升格。
- 起点城市和终点城市没有固定形状，玩法基本盘不稳定。
- 调试时很难判断问题来自 route 参数、素材模板，还是锚点对齐。

## 新生成骨架

### 1. 固定起点区

地图左下角保留一个固定 5x5 起点设施。

建议坐标：

```text
外圈 5x5：墙
内圈 3x3：地板
Box.Core：中心点 (3, 3)
Player：靠近核心箱，例如 (2, 2)
```

如果地图坐标从左下角开始，起点区可以占用：

```text
x = 1..5
y = 1..5
```

其中：

```text
Box.Core = (3, 3)
Player   = (2, 2)
```

### 2. 固定终点区

终点区也使用固定 5x5 设施。

`Target.Core` 位于终点区中心。

终点区位置由 route 侧传入的曼哈顿距离和斜率推导，但一旦确定，不再因为素材拼贴而变动。

终点区同样需要：

```text
外圈 5x5：墙
内圈 3x3：地板
Target.Core：中心点
```

### 3. 中间城市

起点区和终点区确定之后，再在地图中间放置若干矩形城市。

中间城市的规则：

- 每个城市占用一个矩形区域。
- 城市矩形之间不重叠。
- 城市矩形不能覆盖起点区和终点区。
- 城市内部可以贴入 `LevelCollageSourceDatabase` 里的素材地图。
- 素材中的 `Box + Target` 同格继续执行抵消。
- 素材中的普通箱子仍可作为奖励箱或干扰箱。
- 素材不得生成或覆盖 `Box.Core` / `Target.Core`。

第一版先不考虑城市之间如何联通。

## 建议流程

```text
1. 读取 route 参数
   - seed
   - manhattanDistance
   - logSlope
   - difficultyOffset
   - collage constraints

2. 创建空白地图
   - 默认全墙
   - 地图尺寸足够容纳起点区、终点区和中间城市

3. 写入起点区
   - 挖 5x5 设施
   - 内 3x3 地板
   - 中心放 Box.Core
   - 附近放 Player

4. 写入终点区
   - 挖 5x5 设施
   - 内 3x3 地板
   - 中心放 Target.Core

5. 规划中间城市矩形
   - 根据 manhattanDistance 决定城市数量
   - 在起点区和终点区之间采样矩形
   - 做不重叠检测
   - 不满足则重试有限次数

6. 贴入素材
   - 从 LevelCollageSourceDatabase 按约束抽取模板
   - 将模板贴入城市矩形
   - 抵消 Box + Target 同格
   - 清理核心区内的非核心 tag

7. 后续阶段
   - 修路 / 联通
   - 控制奖励箱总量
   - 放置敌人或刷怪点
```

## 核心数据结构

建议生成器内部显式区分三类区域：

```csharp
private sealed class FixedEscortZone
{
    public RectInt Bounds;
    public Vector2Int Center;
}

private sealed class CityRect
{
    public RectInt Bounds;
    public LevelData Template;
}

private sealed class EscortLayout
{
    public FixedEscortZone StartZone;
    public FixedEscortZone TargetZone;
    public List<CityRect> Cities;
}
```

这样可以避免核心点和城市贴图互相污染。

## 验收标准

第一阶段完成后，生成出的押送关卡应满足：

- 必定存在且仅存在一个 `Box.Core`。
- 必定存在且仅存在一个 `Target.Core`。
- `Box.Core` 固定在起点 5x5 区域中心。
- `Target.Core` 固定在终点 5x5 区域中心。
- 玩家固定出生在 `Box.Core` 附近。
- 中间城市矩形互不重叠。
- 中间城市不覆盖起点区和终点区。
- 旧素材不会决定核心点位置。
- 暂时不要求起点、城市、终点全部联通。

## 后续扩展

完成固定核心区之后，再继续补：

- 城市之间的曼哈顿走廊。
- 按路线形状选择横向、纵向或折线路径。
- 约束最终有效奖励箱数量。
- 让终点区附近形成防守压力。
- 按 route 难度调整城市数量、敌人数量、奖励箱数量。
