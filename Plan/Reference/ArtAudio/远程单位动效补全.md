# 明日 远程单位动效补全

优先级：P1

原因：炮、砲、弩手、红中咒术师都是远程/特殊威胁单位。当前逻辑已经能跑，但缺少专属动效会让玩家难以快速理解“谁在威胁我、从哪里打来、为什么受伤/被锁”。这直接影响战斗可读性和产品感。

## 当前状态

已有基础：

- `IntentSystem` 会执行敌人 `AttackIntent`
- `EnemyIntentOverlaySystem` 能画敌人意图范围/路径
- `DrawSystem` 有 beat motion、attack lunge、spawn rise
- `AttackSystem` 命中后会发 `StageEventType.EntityDamaged`
- `StageBeatAudioSystem` 已经改成命中时播放 hit 音效

缺口：

- 远程单位没有发射表现
- 弩手/炮兵/掷弹兵/咒术师攻击看起来容易像“原地闪一下”
- 红中咒术师的“锁定/控制”语义还缺可视化

## 设计原则

- 不做复杂骨骼动画
- 不做大量粒子
- 优先用现有 GridOverlay / DrawSystem / primitive 特效
- 动效必须短，贴合 beat
- 动效表达路径与结果：从谁 -> 到谁/哪格 -> 是否命中

## 单位动效方向

### Crossbow / 弩手

视觉母题：尖底 Rook / 弩塔。

需要表达：直线射击、精准、冷色。

动效方案：

- 攻击时从弩手到目标格画一条细蓝白线束。
- 线束持续一个 beat 的前半段。
- 命中时目标格闪一下蓝白小冲击。
- 如果没有命中，不播放 hit 冲击，只播放短线束或弱化预览。

最小实现：

- `StageEventType.BeforeIntentExecute` + `AttackIntent`
- 如果 actor 是 Crossbow，则画一次 directional beam overlay。

### Grenadier / 掷弹兵 / 砲

视觉母题：中国象棋 `砲`，石字旁，抛射单体。

需要表达：抛物线/投石、单体砸击。

动效方案：

- 发射时从砲棋子上方弹出一个小圆球/石弹。
- 小圆球沿简化弧线飞到目标格。
- 命中时目标格出现小橙色冲击圈。
- 如果没命中，石弹可落空消失，不播 hit。

最小实现：

- 可以先不做真实弧线 mesh，先用橙色点状 path 或短暂圆形 overlay 表达。
- 优先保证“从砲到目标”的方向感。

### Artillery / 炮兵 / 炮

视觉母题：中国象棋 `炮`，火字旁，重火力。

需要表达：重、危险、范围/高压。

动效方案：

- 攻击前炮兵棋子轻微后坐/震动。
- 从炮兵到目标格画粗红橙 beam 或炮弹轨迹。
- 目标格出现更大的火色冲击圈。
- 如果后续有范围伤害，范围格子同步闪烁。

最小实现：

- 和掷弹兵共用 projectile/beam 机制，但颜色、宽度、冲击大小不同。
- 炮兵效果比砲更慢、更重。

### CurseCaster / 红中咒术师

视觉母题：红中麻将牌，两个 `中` 可串联成锁链/封印 icon。

需要表达：锁定、诅咒、控制核心箱子。

动效方案：

- 当咒术师锁定 core box 时，在 caster 和 core box 之间显示紫色连接线/锁链。
- 连接线不是一次性攻击，而是持续状态提示。
- core box 上可显示紫色环或锁定 marker。
- 解除锁定时连接线消失。

最小实现：

- 优先做持续连接线，而不是攻击瞬间特效。
- 可挂在 `EnemyAutoAISystem` 的 curse lock 状态，或单独由 overlay 系统读取锁定集合。

## 技术落点候选

### 方案 A：扩展 DrawSystem

优点：

- 已经订阅 `BeforeIntentExecute / EntityDamaged`
- 已经管理 beat slot
- 可以和单位 motion 时间对齐

缺点：

- DrawSystem 已经偏大，继续塞特效会变重

### 方案 B：新建 StageProjectileFxSystem

优点：

- 远程特效职责清晰
- 可单独维护弩线、炮弹、咒术锁链

缺点：

- 需要接 EventBus，管理生命周期

建议：优先方案 B。`DrawSystem` 继续管单位本体，`StageProjectileFxSystem` 管远程表现。

## 明日执行顺序

1. 先做 Crossbow 线束，因为最能提升远程可读性
2. 再做 炮 / 砲 的共享 projectile/冲击表现
3. 最后做红中咒术师持续锁链

## 验收标准

- 玩家能在一眼内看出弩手正在直线射击
- 炮兵和掷弹兵的攻击表现明显不同
- 未命中的攻击不播放命中冲击
- 命中音效和命中特效在语义上对齐
- 红中锁定 core box 时，玩家能明确看到锁定关系
- 特效不遮挡格子、路径和手牌 UI

## 暂不做

- 骨骼动画
- 复杂粒子系统
- 屏幕震动
- 大爆炸遮挡
- 每个敌人的独立动画状态机
