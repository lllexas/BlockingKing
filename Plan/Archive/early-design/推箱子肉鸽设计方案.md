# 推箱子肉鸽 (Sokoban Roguelike) — 设计方案

## 核心设计哲学

```
肉鸽 ≠ 随机生成关卡
肉鸽 = RPG要素驱动的重复可玩性
        ↓
推箱子里 RPG 要素从哪来？
   玩家在推箱子里的核心诉求 = 比步数（竞速/最优解）
        ↓
步数 = 压力来源（HP/体力）
卡牌 = 破坏规则的杠杆（RPG成长）
        ↓
步数约束 × 卡牌构筑 = 推箱子肉鸽
```

---

## 一句话定位

> 你在玩推箱子，每一步都消耗步数。你的牌库决定了你每一步的"效率"。
> 没有牌的推箱子是硬走，有牌的推箱子是一步当三步用。
> 步数越紧，你越需要一张暴力的牌来打破规则。

---

## 资源体系

```
关卡资源池：
┌─────────────────────────────────────────┐
│  步数上限（比如 30 步） — 硬约束          │
│  归零 = 本关失败，但 Run 继续（扣血）       │
│                                         │
│  每步操作消耗 1 步：                      │
│    ├── 基础移动/推动（↑↓←→）→ 不耗牌      │
│    └── 打出卡牌              → 耗牌但高效  │
│                                         │
│  牌库 — 杀戮尖塔模式                      │
│    ├── 开局摸 N 张手牌                    │
│    ├── 出牌后进弃牌堆                     │
│    ├── 牌库打空 → 洗弃牌堆 → 重新抽        │
│    └── 构筑核心：过关选牌、删牌、升级牌     │
│                                         │
│  道具 — 一次性消耗品（杀戮尖塔药水位）       │
│    ├── 穿墙药、传送药、重置药、手牌刷新药    │
│    └── 稀有掉落，关键时刻翻盘用             │
└─────────────────────────────────────────┘
```

## 卡牌设计：国际象棋式位移

基础移动（↑↓←→）是每个人都有的，**卡牌是打破常规的暴力手段**。

### ♞ 骑士系 — 走日

| 卡牌 | 效果 | 稀有度 |
|------|------|--------|
| **骑士跳** | 走"日"字，无视路径箱子/墙壁，落在目标格 | 普通 |
| **双骑士** | 连续跳两个"日" | 稀有 |
| **回马枪** | 跳到一个箱子后面，并把它反推一格 | 稀有 |
| **骑士冲锋** | 走"日"后若落点旁有箱子，自动推动该箱子 | 稀有 |

### ♝ 主教系 — 斜线

| 卡牌 | 效果 | 稀有度 |
|------|------|--------|
| **主教行** | 沿对角线滑动到撞墙/撞箱为止 | 普通 |
| **斜线推** | 沿对角线方向把面前箱子推一格 | 普通 |
| **对角线换** | 与你对角线方向的箱子交换位置 | 稀有 |
| **十字分割** | 以你为中心，四个斜角方向的箱子各被推一格 | 史诗 |

### ♜ 城堡系 — 直线

| 卡牌 | 效果 | 稀有度 |
|------|------|--------|
| **车行** | 沿直线滑动直到撞墙 | 普通 |
| **冲车** | 沿直线推动一整排箱子一起移动 | 稀有 |
| **贯穿** | 直线冲过 1 个箱子，落在箱子后面 | 稀有 |
| **长矛** | 直线方向隔一格推动一个箱子 | 普通 |

### ♛ 皇后系 — 终极

| 卡牌 | 效果 | 稀有度 |
|------|------|--------|
| **皇后驾到** | 向任意方向移动到撞墙（车+主教合体） | 史诗 |
| **君临** | 向任意方向推动 1 个箱子到撞墙 | 史诗 |
| **棋局终了** | 所有箱子往最近的目标点位移 1 格 | 传说 |

### 🧪 道具（药水位，一次性消耗品）

| 道具 | 效果 |
|------|------|
| **穿墙药** | 无视墙壁行走 1 步 |
| **传送药** | 瞬移到任意地板格 |
| **重置药** | 本关步数回退 5 步 |
| **洗牌药** | 立即将所有弃牌堆洗回手牌 |
| **换位药** | 与任意一个箱子交换位置 |

## 步数杠杆的经济学

**核心逻辑：卡牌的价值 = 你省了多少步**

```
基础移动：1 步 = 1 格位移   → 杠杆率 1x
骑士跳：  1 步 = 走日省5步  → 杠杆率 ~5x
冲车：    1 步 = 推整排     → 杠杆率 Nx（N=箱子数）
皇后驾到：1 步 = 任意位移   → 杠杆率 10x+
```

```
关卡难度递增曲线：
  第1关：步数 40，对手牌要求低 → 基础走也能过
  第3关：步数 25，需要骑士跳等卡牌提效
  第5关：步数 18，需要稀有卡牌 combo
  BOSS关：步数 10，必须有完整的构筑才够
```

**爽感来源：**
- 前期没有牌，每一步都是"硬走"
- 中期拿到了骑士跳，发现原来 5 步的路 1 步走完 → **爽**
- 后期构筑成型，冲车 + 皇后 combo 一波清场 → **巨爽**

## 集成战略式肉鸽框架

### 探索地图

```
            ┌── 战斗 ── 奖励 ── 战斗 ── BOSS
           │
  起点 ─── ┼── 紧急战斗 ── 商店 ── 事件 ──
           │
            └── 事件 ── 休息 ── 战斗 ── 精英
```

各节点类型：

| 节点 | 内容 |
|------|------|
| ⚔️ **战斗** | 一局推箱子解谜，固定步数上限 |
| 🔥 **紧急战斗** | 战斗 + 额外限制（步数更少/箱子更多/特殊地形） |
| 🎯 **奖励** | 三选一：获得一张新卡 / 升级一张已有卡 / 获得一个道具 |
| 🏪 **商店** | 消耗"步数结余"购买卡牌或道具 |
| ❓ **事件** | 随机选择分支，有正面/负面结果 |
| 🛌 **休息** | 回满步数（相当于回血） |
| 👑 **BOSS** | 超难关卡，步数极紧，考验构筑质量 |

### Run 生命周期

```
[开始 Run]
   │ 初始牌库：骑士跳×1、车行×1、基础移动（永久免费）
   ▼
[第1关 战斗] → 步数35 → 用基础移动就能过
   │ 奖励：三选一【主教行 / 回马枪 / 斜线推】
   ▼
[第2关 战斗] → 步数30 → 试试新牌
   │ 商店：花步数买【贯穿】
   ▼
[第3关 紧急战斗] → 步数20！开始有压力了
   │ 奖励：升级【骑士跳 → 双骑士】
   ▼
[事件] → "看到一面裂缝的墙" → 穿过去？→ 获得【穿墙药】
   ▼
[第4关 战斗] → 步数22
   ▼
[第5关 BOSS] → 步数12！！！构筑大考
   │ 构筑成型：骑士跳×1 升级为双骑士 + 冲车 + 皇后驾到
   │ 解法：冲车清线 → 双骑士绕后 → combo通关
   ▼
通关 or 步数耗尽 → Game Over → 从头再来
```

## 7天开发计划

### Day 1 — 核心框架 + 基础推箱子
- 搭建 Unity 项目，网格渲染系统
- **玩家移动 + 箱子推动逻辑**（核心物理）
- 关卡加载（从 `LevelData` / int[][] 渲染关卡）
- 目标点判定 + 通关检测
- 步数统计显示

### Day 2 — 卡牌系统核心
- **牌库（Deck）数据结构**：List<CardSO>
- **手牌管理**：抽牌、出牌、弃牌、洗牌（杀戮尖塔完整机制）
- 卡牌基类 `CardSO` + 卡牌效果接口
- 实现第一批卡牌：骑士跳、车行、主教行
- UI：手牌展示区、出牌交互

### Day 3 — 完整卡牌池 + 道具系统
- 实现所有卡牌（骑士系×4、主教系×4、城堡系×4、皇后系×2）
- 道具系统（穿墙药、传送药、重置药、洗牌药、换位药）
- 卡牌稀有度机制（普通/稀有/史诗/传说）
- 卡牌动画效果（DOTween 平滑表现）

### Day 4 — 关卡生成 + 难度曲线
- **手搓 10~15 个关卡**，覆盖从入门到 BOSS
- 关卡按难度分级，对应不同 Run 阶段
- 步数上限的数值配表
- 紧急战斗/特殊关卡的变体配置

### Day 5 — 集成战略式肉鸽框架
- **节点地图系统**：路线选择 UI
- 节点类型实现：战斗、紧急战斗、奖励、商店、事件、休息、BOSS
- 节点之间的状态传递（奖励给卡牌、商店扣步数等）
- Run 生命周期管理（开始、进行中、结束）

### Day 6 — UI 全面打磨
- 主菜单（开始 Run、图鉴、设置）
- 卡牌收藏界面（查看已解锁/未解锁卡牌）
- 关卡内 HUD（步数、手牌、道具栏）
- BOSS 关特殊演出
- 通关/失败界面 + 本局统计数据

### Day 7 — 测试、平衡、打包
- 全流程跑测（从开始到 BOSS 的完整 Run）
- 数值平衡（卡牌强度、步数上限、掉落概率）
- BUG 修复
- 打包 WebGL / Windows

## 风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 卡牌效果与推箱子物理的交互 bug | 中 | 每张卡单独测试 + 组合测试 |
| 7 天做不完所有卡牌 | 中 | 优先实现骑士系+城堡系 (Day2-3)，主教系和皇后系是加分项 |
| 数值不平衡 | 低 | 手搓关卡可以单独调步数上限适配 |
| 肉鸽地图节点系统复杂 | 中 | 先用最简单的线性链（战斗→奖励→战斗→BOSS），后续再加分支 |
