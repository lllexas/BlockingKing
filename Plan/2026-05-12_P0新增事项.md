# 2026-05-12 P0 新增事项

日期：2026-05-12

这页只记录今天新增、且必须优先处理的 P0 项。

## P0 1. 牌加入牌库时的动画

目标：
- 牌在进入牌库时要有明确的视觉反馈。
- 不能只是数据上加入，画面上必须让玩家看见“这张牌进库了”。

关注点：
- 动画要和现有卡牌 UI 结构兼容。
- 不要破坏抽牌、弃牌、洗牌的现有节奏。
- 需要有明确的进入、停顿、归位或收束表现。

## P0 2. 新游戏教程

目标：
- 新开局时提供可理解的基础教学。
- 让第一次接触的人知道怎么移动、怎么推箱、怎么理解敌人和目标。

关注点：
- 教程必须建立在稳定的基础体验之上。
- 先保证流程、表现、结算稳定，再做细教程。
- 教程入口、跳过、恢复正式流程都要可控。

## P0 3. 给敌人补齐攻防数值

目标：
- 敌人不再只有行为，没有清晰数值。
- 攻击、防御、生命这类基础战斗参数要完整填好。

关注点：
- 不同敌人要有差异，不要一套模板糊过去。
- 数值要和关卡阶段、角色定位、远程/近战职责匹配。
- 填完后要能直接支撑平衡调整。

## P0 4. 检查敌人身上的字符不显示

目标：
- 解决敌人身上字符/文字不显示的问题。
- 保证敌人标识、状态字样或相关提示能稳定出现。

关注点：
- 先查渲染链路和材质/字体设置。
- 再查遮挡、层级、Camera、Canvas 或 DrawSystem 的显示条件。
- 这是可读性问题，影响第一眼识别，优先级必须高。

## P0 5. classic 关卡抽取口径错误

目标：
- classic 模式在关卡抽取时，`box` 范围必须按【全部箱子数量】判断。
- 不能继续沿用 escort 那套【有效箱子数量】口径。

问题描述：
- 现在的 feature filter 里有一个 box 数量范围。
- escort 会把开局位于目标点的箱子删掉，所以它应当使用有效箱子数。
- classic 不会做这件事，必须使用总箱子数，否则会错抽关卡。

## 备注

这些都按 P0 处理。不要在它们之间插入新的 P1/P2 工作，先把核心体验收住。
